> 分析日期：2026-06-11
> 分析范围：全部源码（src/ 下 19 个文件，约 18,500 行 JS）

---

## 一、项目概述

基于 **Three.js + Vite** 的浏览器端 3D 几何构造工具。用户通过文本命令（如 `A = Point(0,0,0)`、`Segment(A,B)`）和鼠标交互创建几何对象，支持点、线段、直线、射线、三角形、多边形、平面，以及中点、折叠、反射、距离等计算操作。

**技术栈：** Vanilla JS (ES Modules) + Three.js 0.165 + Vite 5.4，无框架、无 TypeScript。

**架构特点：**
- 自定义词法分析器 + 递归下降解析器 + 命令执行器的完整命令管线
- EventBus 事件驱动的模块解耦
- 依赖图管理对象间的父子关系，拖拽点时自动级联更新
- HistoryManager 管理撤销/重做栈

## 二、确认的 Bug（经源码逐行验证）

### 🔴 高严重度

#### Bug #1：`ndcToWorld` 除零错误

- **文件：** `src/main.js:479`
- **代码：**
  ```js
  const distance = -camera.position.z / dir.z;
  ```
- **问题：** 当相机视线平行于 XY 平面时，`dir.z` 为 0 或接近 0，导致 `distance` 为 `Infinity` 或 `NaN`，后续 `pos` 计算产生垃圾值。
- **触发场景：** 用户在画点模式下，将相机旋转到接近水平视角后点击空白处。
- **修复建议：** 添加 `Math.abs(dir.z) < 1e-6` 的守卫，回退到默认距离或拒绝操作。

---

#### Bug #2：删除对象后点名冲突

- **文件：** `src/main.js:298`
- **代码：**
  ```js
  const pointName = `P${store.size() + 1}`;
  ```
- **问题：** 若 P1、P2、P3 存在，删除 P2 后 `store.size()` 为 2，下一个点名 `P3` 与已存在的 P3 冲突。`store.register()` 抛出 `"对象 "P3" 已存在"` 异常。
- **同类问题：** `src/parser/executor.js:436` 的 `cmdRegularPolygon` 中：
  ```js
  const name = `${prefix}_${this.store.size() + 1}`;
  ```
  同样存在命名冲突。而 `cmdEquilateralTriangle`（line 492-494）使用 `baseIndex + i` 稍好，但仍可能与已有名称冲突。
- **对比：** `generateAutoName()`（executor.js:107-122）使用 while 循环查找未用名称，是安全的。
- **修复建议：** 使用递增计数器（不因删除而回退）或复用 `generateAutoName()` 的 while 循环模式。

---

#### Bug #3：平面方程 `c=0` 时除零

- **文件：** `src/render/geomRenderer.js:238`
- **代码：**
  ```js
  point = new THREE.Vector3(0, 0, -d / c);
  ```
- **问题：** 当平面方程 `ax + by + cz + d = 0` 中 `c = 0`（平面垂直于 XY 平面）时，`-d / c` 产生 `Infinity`，导致渲染崩溃。
- **触发场景：** `Plane(1, 0, 0, 5)` 或任何法向量在 XY 平面内的平面。
- **修复建议：**
  ```js
  if (Math.abs(c) > 1e-6) point = new THREE.Vector3(0, 0, -d / c);
  else if (Math.abs(b) > 1e-6) point = new THREE.Vector3(0, -d / b, 0);
  else point = new THREE.Vector3(-d / a, 0, 0);
  ```

### 🟡 中严重度

#### Bug #4：线型对象选中高亮无效

- **文件：** `src/interaction/selector.js:76-79`
- **代码：**
  ```js
  const highlightMat = mesh.material.clone();
  highlightMat.emissive = new THREE.Color(0x55ccee);
  highlightMat.emissiveIntensity = 0.5;
  ```
- **问题：** 线段、直线、射线使用 `LineBasicMaterial`，该材质没有 `emissive` 属性。设置后静默无效，选中线型对象无任何视觉反馈。
- **影响：** 只有点（`MeshStandardMaterial`）和面（`MeshStandardMaterial`）有高亮效果。
- **修复建议：** 对线型对象使用颜色叠加或 `LineDashedMaterial` 虚线变化来表示选中状态。

#### Bug #5：`linewidth` 被 WebGL 忽略

- **文件：** `src/render/geomRenderer.js:154`
- **代码：**
  ```js
  linewidth: obj.style.lineWidth || 2
  ```
- **问题：** WebGL 规范中 `linewidth` 在 Windows（ANGLE）和 macOS 上始终为 1px，此设置无效。
- **影响：** 用户无法看到或调整线条粗细。
- **修复建议：** 使用 Three.js 的 `Line2` / `LineMaterial`（`three/addons/lines/`）实现可变宽度线条。

#### Bug #6：反馈消息竞态

- **文件：** `src/main.js:89-96`
- **代码：**
  ```js
  feedback.textContent = `✓ 执行成功`;
  feedback.className = 'success';
  setTimeout(() => {
      feedback.textContent = '';
      feedback.className = '';
  }, 3000);
  ```
- **问题：** 若 3 秒内执行第二条命令，第一个 `setTimeout` 回调会清除第二条命令的成功消息。
- **修复建议：** 在 `setTimeout` 前保存 timer ID 并 `clearTimeout`，或使用闭包捕获当前状态。

#### Bug #7：属性面板 XSS 风险

- **文件：** `src/main.js:120-121`
- **代码：**
  ```js
  html += `<div class="prop-row"><label>名称</label><input type="text" value="${obj.name}" readonly></div>`;
  ```
- **问题：** `obj.name` 直接插值到 `innerHTML` 中，未转义。虽然当前名称由程序生成（P1、seg1 等），但若用户通过某种方式注入含 HTML 的名称，可触发 XSS。
- **实际风险：** 较低。当前系统中对象名称不可直接由用户自由输入。
- **修复建议：** 使用 `textContent` 或转义函数处理插值。

#### Bug #8：切换投影重置相机位置

- **文件：** `src/render/scene.js:293-295`
- **代码：**
  ```js
  this.camera.position.set(10, 5, 8);
  this.camera.lookAt(0, 0, 0);
  this.camera.up.set(0, 0, 1);
  ```
- **问题：** 按空格切换透视/正交投影时，相机始终重置到 `(10, 5, 8)`，用户当前视角丢失。
- **修复建议：** 在切换前保存当前相机位置/旋转，切换后恢复。

#### Bug #9：凹多边形三角化错误

- **文件：** `src/render/geomRenderer.js:304-311`
- **代码：**
  ```js
  const center = points[0];
  for (let i = 1; i < points.length - 1; i++) {
      vertices.push(
          center.x, center.y, center.z,
  ```

- **文件：** `src/parser/executor.js:213`
- **代码：** `case 'unary': return -this.evaluate(node.operand);`
- **问题：** 若对点对象取负（如 `-A`），返回 `NaN`，无错误提示。

#### Bug #12：平面反射缺少 `compute` 函数

- **文件：** `src/parser/executor.js:611-616`
- **问题：** `cmdReflect` 中线反射有 `compute` 函数（line 587-595），但平面反射没有。移动平面或源点时，反射点不会自动更新。
- **修复建议：** 为平面反射添加 `compute` 函数，与线反射保持一致。

#### Bug #13：`unregister` 未清理子对象的父引用

- **文件：** `src/geometry/store.js:79-83`
- **问题：** 删除对象 A 时，只从 A 的父对象的 `dependents` 集合中移除了 A。但依赖 A 的子对象的 `parents` 数组仍包含 A 的名称。若 A 的名称被新对象复用，旧子对象会错误地成为新对象的依赖。
- **修复建议：** 在 `unregister` 中遍历 `dependents.get(name)`，从每个子对象的 `parents` 集合中移除被删对象。

---

#### Bug #14：`revokeObjectURL` 过早调用

- **文件：** `src/storage.js:91-93`
- **代码：**
  ```js
  link.click();
  URL.revokeObjectURL(url);
  ```
- **问题：** `link.click()` 触发异步下载，`revokeObjectURL` 在其后同步调用。大多数浏览器下能正常工作，但在慢浏览器/大文件场景下，URL 可能在下载开始前被释放。
- **修复建议：** 使用 `setTimeout(() => URL.revokeObjectURL(url), 1000)` 延迟释放。

#### Bug #15：`deserialize` 触发大量自动保存

- **文件：** `src/geometry/store.js:215-219`
- **问题：** `deserialize` 先调用 `clear()`（触发 `store:cleared` 事件），再对每个对象调用 `register()`（各触发 `object:created` 事件）。每次事件都触发 `StorageManager.scheduleSave()`。虽然有 2 秒节流，但仍造成不必要的定时器创建和取消。
- **修复建议：** 添加 `batchMode` 标志，批量操作期间暂停事件发送，操作完成后统一触发一次。

#### Bug #16：`Distance` 结果赋值给变量创建点

#### Bug #17：注册对象时未验证父对象存在性

- **文件：** `src/geometry/store.js:56-61`

#### Bug #18：命令历史导航 `prev()` 边界错误

- **文件：** `src/main.js:51-56`
- **代码：**
  ```js
  prev() {
      if (this.index > 0) {
          this.index--;
          return this.history[this.index];
      }
      return this.history[0] || '';
  }
  ```
- **问题：** 当 `index === history.length` 时按 Up 键，`index > 0` 为 true，`index` 减到 `length - 1`，返回最后一条 — 这是正确的。但若已在 `index === 0`（最老的命令）再按 Up，`index > 0` 为 false，返回 `history[0]` — 也是正确的。实际上这个逻辑在正常场景下工作正常。
- **修正：** 原报告称"跳过最后一条"经验证不准确。`prev()` 的边界行为在常规使用下是正确的。

#### Bug #22：高亮材质泄漏

- **文件：** `src/interaction/selector.js`
- **问题：** `highlight()` 将原始材质存入 `originalMaterials` Map（line 73），但 `unhighlight()` 只在主动取消选中时调用。当对象被 Delete 命令删除时：

## 三、不是 Bug 的澄清

### ❌ 词法分析器负数处理（已排除）

原报告中提到的"负数处理不完整"**不是 bug**。

- **tokenizer.js:23** 的正则确实将 `-5` 拆分为 `MINUS` 和 `NUMBER:5` 两个 token
- 但 **parser.js:107-113** 的 `parseUnary()` 正确处理了一元负号：
  ```js
  parseUnary() {
      if (this.check(TokenType.MINUS)) {
          this.advance();
          const operand = this.parseUnary();
          return { type: 'unary', op: '-', operand };
      }
      return this.parsePrimary();
  }
  ```
- **executor.js:212-213** 正确执行一元负号：`return -this.evaluate(node.operand);`
- 因此 `Point(-1, 2, 3)` 和 `Fold(P, L, -90)` 都能正确解析执行。

## 四、架构层面改进建议

### 4.1 工程基础设施

| 缺失项 | 建议 |
|--------|------|
| 类型系统 | 引入 TypeScript，几何计算密集型代码极易因类型错误产生 bug |
| 测试 | 添加 Vitest 单元测试，至少覆盖 tokenizer、parser、executor、calc |
| Linting | 添加 ESLint + Prettier，统一代码风格 |
| CI/CD | GitHub Actions 自动化构建和测试 |

### 4.2 功能改进

| 方向 | 建议 |
|------|------|
| 凹多边形 | 使用 ear clipping 算法替代当前的扇形分割 |
| 线条粗细 | 使用 `THREE.Line2` 替代 `THREE.Line` |
| 撤销/重做 | 批量事件或快照机制，避免级联重渲染 |
| 变量系统 | `Distance` 等数值结果应存储为标量 |
| 标签更新 | 使用 `updateLabelPosition` 替代销毁-重建 |

### 4.3 安全改进

| 问题 | 建议 |
|------|------|
| XSS | `innerHTML` 改用 `textContent` 或转义函数 |
| 孤立依赖 | `node_modules` 中有未声明的 `better-sqlite3`，运行 `npm prune` 清理 |

### 4.4 文档维护

- 更新 README，移除不存在的文件引用
- 与实际代码结构保持同步
