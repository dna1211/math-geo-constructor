# 3D 几何构造工具 -- 代码分析报告

## 一、已确认的 BUG

### BUG-1: 正多边形顶点偏移错误 (严重)

**文件**: `src/geometry/construct.js` 第 68-92 行

第一个顶点直接使用 `from` 端点，但后续顶点以线段中点 `mid` 为圆心展开。对于正三角形，`from` 到 `mid` 的距离是 `edgeLength/2`，而外接圆半径是 `edgeLength / sqrt(3) ≈ 0.577 * edgeLength`，两者不相等，导致顶点位置错误。

---

### BUG-2: EquilateralTriangle 命名冲突 (严重)

**文件**: `src/parser/executor.js` 第 492 行

```javascript
const name = `v3_${this.store.size() + 1}`;
```

循环中每次调用 `store.size()` 返回相同的值，三个顶点会全部使用同一个名称，导致"对象已存在"错误。对比 `cmdRegularPolygon` 的写法是正确的。

---

### BUG-3: 撤销旧值获取错误 (严重)

**文件**: `src/geometry/history.js` 第 50-69 行

`recordUpdate` 在 `bus.on('object:updated', ...)` 回调中被调用，但此时 `store.update()` 中的 `Object.assign(obj.data, changes.data)` 已经执行完毕，`obj.data` 已经是更新后的值，无法正确恢复旧状态。

---

### BUG-4: undo/redo 事件递归 (中等)

**文件**: `src/geometry/history.js` 第 77-101 行

`undo()` 调用 `store.unregister()`/`store.register()`/`store.update()` 时，这些方法内部 emit 事件，而 `HistoryManager` 监听了这些事件并自动记录新历史条目，形成无限递归式的状态污染。

---

### BUG-5: Tokenizer 减号/负号歧义 (中等)

**文件**: `src/parser/tokenizer.js` 第 22 行

正则 `/\s*([A-Za-zͰ-Ͽ_][A-Za-z0-9Ͱ-Ͽ_']*|-?\d+\.?\d*|[=(),+\/*-])/g` 中 `-` 同时匹配为减号和负数的一部分。边界情况如 `Point(A + -5, 0, 0)` 可能导致解析异常。

---

### BUG-6: Polygon 渲染只使用前两个坐标分量 (中等)

**文件**: `src/render/geomRenderer.js` 第 289-321 行

使用 `THREE.Shape`（2D）构建多边形，只使用 x 和 y 坐标。如果多边形顶点不在同一 z 平面上，渲染结果将被压平。

---

### BUG-7: reflectPlane 除以零未处理

**文件**: `src/geometry/calc.js` 第 102 行

如果平面参数 `a, b, c` 全为零，分母为零导致返回 NaN。

---

### BUG-8: Plane 方程形式永远无法使用 (严重)

**文件**: `src/parser/executor.js` 第 31 行

`Plane` 命令注册为 `{ minArgs: 1, maxArgs: 3 }`，但 `cmdPlane` 中检查 `args.length === 4`。参数为 4 时会被 `executeCall` 的参数校验拒绝。

---

## 二、潜在问题

| 问题 | 位置 | 说明 |
|------|------|------|
| resize 事件未清理 | `scene.js` L21, `labelRenderer.js` L27 | 热重载时旧监听器不会移除 |
| EventBus 事件泄漏 | `eventBus.js` L58 | 无 `dispose()` 方法注销监听器 |
| getPickableObjects 每帧遍历 | `main.js` L440-448 | 大量对象时影响性能 |
| 拖拽时每帧多次重渲染 | `main.js` L451-465 | 性能抖动 |
| renderUpdate 销毁重建 | `geomRenderer.js` L65-68 | GC 压力大 |
| linewidth 不生效 | `geomRenderer.js` 多处 | WebGL 限制，linewidth>1 被忽略 |
| toggleProjection 状态残留 | `scene.js` L273-302 | 切换后视角可能跳变 |
| exportPNG 无透明通道 | `scene.js` L257-270 | 背景是黑色 |
| 标签偏移固定 | `geomRenderer.js` L129 | 旋转后可能被遮挡 |

## 三、改进方案

| 优先级 | 问题 | 修复方案 |
|--------|------|----------|
| P0 | BUG-3 | 在 `store.update()` 中修改数据前先 emit `object:beforeUpdate` 事件 |
| P0 | BUG-4 | 添加 `_isUndoRedoing` 标志位，undo/redo 期间跳过历史记录 |
| P0 | BUG-8 | 将 `Plane` 的 `maxArgs` 改为 4 |
| P1 | BUG-2 | 使用循环内递增的 counter 变量 |
| P1 | BUG-1 | 所有顶点统一从圆心+角度计算 |
| P1 | BUG-7 | 添加 epsilon 检查 |
| P2 | 性能 | 拖拽时使用 requestAnimationFrame 节流 |
| P2 | linewidth | 使用 Line2 或 ShaderMaterial 实现粗线 |
| P3 | 清理 | 添加 `dispose()` 方法 |
