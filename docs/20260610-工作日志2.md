## 一、Bug 修复

### 1.1 代码分析

对整个项目进行了全面代码分析，发现 8 个 bug 和 10 个潜在问题，生成了详细的分析报告。

### 1.2 P0 级修复

**BUG-3: 撤销旧值获取错误**
- 在 `store.update()` 中修改数据前先记录旧值
- 通过事件传递 `beforeData` 和 `beforeStyle` 给 `HistoryManager`

**BUG-4: undo/redo 事件递归**
- 添加 `_isUndoRedoing` 标志位
- undo/redo 期间跳过历史记录

**BUG-8: Plane 方程形式无法使用**
- 将 `Plane` 命令的 `maxArgs` 从 3 改为 4

### 1.3 P1 级修复

**BUG-2: EquilateralTriangle 命名冲突**
- 使用循环内递增的索引生成唯一顶点名称

**BUG-1: 正多边形顶点偏移错误**
- 统一从圆心+角度计算所有顶点

**BUG-7: reflectPlane 除以零未处理**
- 添加 epsilon 检查，当平面法向量长度接近零时返回原点

### 1.4 P2 级修复

**BUG-5: Tokenizer 减号/负号歧义**
- 移除正则中的负数匹配，减号统一作为单独的 MINUS token

**BUG-6: Polygon 渲染只使用前两个坐标分量**
- 改用 BufferGeometry 三角扇构建 3D 多边形

### 1.5 交互问题修复

**问题1：选择一个点后其他点不能选中**
- 在 `createPoint` 中添加 `mesh.userData.objectType = 'point'`
- 使 picker 的包围球检测能正确识别点对象

**问题2：拖拽点时关联线不动**
- 在 `store.update()` 中自动遍历依赖对象并触发 `object:updated` 事件
- 添加 `_fromParent` 标志避免依赖对象更新被记录到历史栈

**问题3：计算型对象（Midpoint等）不跟随父对象移动**
- `types.js`: 对象添加 `compute` 字段支持重算函数
- `executor.js`: Midpoint、Fold、Reflect 创建时附带 `compute` 函数
- `store.js`: register 时传递 `compute` 函数；更新依赖对象时先调用 `compute` 重算数据
