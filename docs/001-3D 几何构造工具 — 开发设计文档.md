## 3D 几何构造工具 — 开发设计文档

---

### 一、项目概述

#### 1.1 目标

构建一个基于命令输入 + 鼠标交互的 3D 几何图形构造工具，用户可以通过命令精确建图，也可以通过鼠标拖拽微调。

#### 1.2 核心能力

```
命令输入建图    点/线/面/体的精确构造
拖拽微调        鼠标拖拽顶点，关联对象自动更新
翻折控制        绕轴任意角度旋转
样式控制        颜色/虚线/透明度
历史管理        撤销/重做
```

#### 1.3 技术栈

```
渲染引擎     Three.js（ES Module）
UI 框架      纯 HTML/CSS/JS（不引入前端框架）
构建工具     Vite（开发服务器 + 构建打包）
数据库       better-sqlite3（开发环境 SQLite 持久化）
外部依赖     Three.js 及其 addons（OrbitControls、CSS2DRenderer）
```

### 四、数据结构设计

#### 4.1 GeomObject（几何对象基类）

```javascript
// 所有几何对象的通用结构
{
    name: 'A',                    // 对象名称（用户定义）
    type: 'point',                // 类型
    data: { ... },                // 类型特有数据
    style: {                      // 样式
        color: '#e0dcd2',
        opacity: 1.0,
        dash: false,
        visible: true,
        label: 'A'
    },
    parents: [],                  // 依赖的父对象名称列表
    renderRef: null               // 对应的 Three.js 对象引用（渲染层绑定）
}
```

#### 4.2 各类型 data 结构

```javascript
// 点
type: 'point'
data: { x: 0, y: 0, z: 0 }

// 线段
type: 'segment'
data: { from: 'A', to: 'B' }       // 引用点名

// 射线
type: 'ray'
data: { origin: 'A', through: 'B' }

// 无限直线
type: 'line'
data: { from: 'A', through: 'B' }

// 三角形
type: 'triangle'
data: { points: ['P', 'A', 'D'] }

// 多边形
type: 'polygon'
data: { points: ['A', 'B', 'C', 'D'] }

// 平面（方程定义）
type: 'plane'
data: { a: 0, b: 0, c: 1, d: 0 }          // ax + by + cz + d = 0
// 或
type: 'plane'
data: { points: ['A', 'B', 'C'] }          // 过三点

// 翻折结果（特殊类型，本质是一组新点）
type: 'fold'
data: {
    source: 'P',                              // 原始点
    axis: { from: 'A', to: 'D' },             // 旋转轴
    angle: 90,                                 // 角度
    result: 'P_prime'                          // 结果点名
}
```

#### 4.3 ObjectStore（对象仓库）

```javascript
{
    // 核心存储
    objects: Map<string, GeomObject>,
    
    // 历史栈
    undoStack: [],               // [{ type, name, snapshot }]
    redoStack: [],
    
    // 依赖图（邻接表）
    dependents: Map<string, Set<string>>,    // A → Set{'seg1', 'polygon1', 'E'}
    parents:    Map<string, Set<string>>,    // E → Set{'A', 'B'}
}
```

#### 4.4 依赖图操作

```javascript
// 注册对象时自动建立依赖
register('E', { type: 'midpoint', parents: ['A', 'B'] })
    → dependents.get('A').add('E')
    → dependents.get('B').add('E')
    → parents.get('E').set('A', 'B')

// 删除对象时断开依赖
unregister('E')
    → dependents.get('A').delete('E')
    → dependents.get('B').delete('E')
    → 同时删除 E 的渲染对象

// 拖拽 A 时，获取所有需要更新的对象
getAffected('A')
    → BFS/DFS 遍历依赖图
    → 返回 ['E', 'seg1', 'polygon1', ...]
    → 按拓扑序更新
```

### 五、命令解析器详细设计

#### 5.1 Tokenizer

**输入：** 字符串
**输出：** Token 数组

```
Token 类型：
    IDENT      标识符   [A-Za-z\u0370-\u03FF][A-Za-z0-9\u0370-\u03FF_]*
    NUMBER     数字     -?\d+(\.\d+)?
    LPAREN     (
    RPAREN     )
    COMMA      ,
    EQUALS     =
    PLUS       +
    MINUS      -
    STAR       *
    SLASH      /
```

**核心正则：**

```javascript
const TOKEN_RE = /\s*([A-Za-z\u0370-\u03FF_][A-Za-z0-9\u0370-\u03FF_']*|-?\d+\.?\d*|[=(),+\/*-])/g;
```

**特殊处理：**
- 跳过空白
- 支持 `P'` 中的撇号作为标识符一部分
- 支持 `α, β, θ` 等希腊字母作为变量名
- 支持负数 `-1.5`

#### 5.2 Parser

**输入：** Token 数组
**输出：** AST 节点

**AST 节点类型：**

```javascript
// 赋值语句
{ type: 'assign', name: 'A', value: ExprNode }

// 函数调用（无赋值）
{ type: 'call', func: 'Segment', args: [ExprNode, ...] }

// 无参命令
{ type: 'command', name: 'Undo' }

// --- 表达式节点 ---

// 标识符引用
{ type: 'ident', name: 'A' }

// 数字
{ type: 'number', value: 0 }

// 坐标元组
{ type: 'tuple', elements: [ExprNode, ExprNode, ExprNode] }

// 二元运算
{ type: 'binary', op: '+', left: ExprNode, right: ExprNode }

// 函数调用表达式
{ type: 'funcCall', func: 'Midpoint', args: [ExprNode, ...] }
```

**解析优先级（从低到高）：**

```
1. 赋值    A = expr
2. 加减    expr + expr, expr - expr
3. 乘除    expr * expr, expr / expr
4. 一元    -expr
5. 调用    Func(args...)
6. 原子    标识符 / 数字 / (元组) / (子表达式)
```

#### 5.3 Executor

**输入：** AST 节点
**输出：** 几何对象变更（创建/修改/删除）

```javascript
function execute(ast, store, renderer) {
    switch (ast.type) {
        case 'assign':
            executeAssign(ast, store, renderer);
            break;
        case 'call':
            executeCall(ast, store, renderer);
            break;
        case 'command':
            executeCommand(ast, store, renderer);
            break;
    }
}
```

**命令注册表（可扩展）：**


### 六、交互系统设计

#### 6.1 工具状态机

```javascript
const ToolState = {
    IDLE:       'idle',        // 默认，可选择/拖拽
    PLACING_POINT: 'placing',  // 画点模式，点击放置
    DRAWING_LINE: 'drawing',   // 画线模式，依次选两点
    DRAWING_POLYGON: 'polygon' // 画面模式，依次选点，双击闭合
};
```

**状态转换：**

```
IDLE ──→ PLACING_POINT    （点击"画点"按钮）
IDLE ──→ DRAWING_LINE     （点击"画线"按钮）
IDLE ──→ DRAWING_POLYGON  （点击"画面"按钮）

PLACING_POINT  ──→ IDLE   （放置一个点后自动返回）
DRAWING_LINE   ──→ IDLE   （选完两点后自动返回）
DRAWING_POLYGON ──→ IDLE   （双击/右键闭合后返回）

任何状态 ──→ IDLE         （按 Esc 键取消）
```

**各状态下的鼠标行为：**

| 状态 | 左键点击空白 | 左键点击对象 | 右键 | 移动 |
|---|---|---|---|---|
| IDLE | 取消选择 | 选中对象 | 无 | 拖拽中则更新 |
| PLACING_POINT | 放置点 | 忽略 | 取消 | 显示预览点 |
| DRAWING_LINE | 忽略 | 记录端点 | 取消 | 显示预览线 |
| DRAWING_POLYGON | 忽略 | 记录顶点 | 闭合 | 显示预览边 |

#### 6.2 拖拽系统

```
mousedown
    ↓
Raycaster 检测是否命中可拖拽对象（点的包围球）
    ↓ 命中
记录拖拽目标 + 拖拽平面（通过该点的 XZ 平面）
    ↓
mousemove（拖拽中）
    ↓
Raycaster 与拖拽平面求交 → 计算新位置
    ↓
更新点坐标 → 触发依赖更新 → 重新渲染关联对象
    ↓
mouseup
    ↓
结束拖拽 → 记入历史栈
```

**拖拽平面选择：**

| 按键 | 拖拽平面 | 用途 |
|---|---|---|
| 无 | XZ 平面（y = 点的 y 值） | 默认，地面移动 |
| Shift | XY 平面（z = 点的 z 值） | 竖直面移动 |
| Ctrl | YZ 平面（x = 点的 x 值） | 竖直面移动 |

#### 6.3 选中与高亮

```javascript
// 选中对象后
selectedObject = obj;

// 高亮方式
- 点：球体变为亮色 + 放大 1.2 倍 + 外发光
- 线：变为亮色 + 线宽增加
- 面：变为亮色 + 边高亮

// 属性面板联动
- 右侧显示该对象的可编辑属性
- 修改属性后实时更新渲染
```

### 七、渲染层设计

#### 7.1 对象 → Three.js 映射

| GeomObject | Three.js 表示 |
|---|---|
| Point | SphereGeometry(r=0.06) + MeshStandardMaterial |
| Segment | BufferGeometry + LineBasicMaterial |
| Segment(dash) | BufferGeometry + LineDashedMaterial |
| Ray | 从 origin 向 through 方向延伸到视口边界 |
| Line | 从 through 方向两端延伸到视口边界 |
| Triangle | 自定义三角面 BufferGeometry + 透明材质 |
| Polygon | 三角化后自定义 BufferGeometry + 透明材质 |
| Plane | 大矩形面（可裁剪）+ 透明材质 |
| Label | CSS2DObject + TextBlock |

#### 7.2 渲染更新策略

```javascript
// 对象创建
function renderCreate(obj) → 创建 Three.js 对象并加入场景

// 对象更新（拖拽/命令修改后）
function renderUpdate(obj) → 更新 Three.js 对象的 geometry/position

// 对象删除
function renderRemove(obj) → 从场景移除并销毁 Three.js 对象

// 样式变更
function renderStyle(obj) → 更新 material 属性（颜色/透明度/虚线）
```

**关键：每个 GeomObject 保存一个 renderRef 指向对应的 Three.js 对象，更新时直接操作。**

### 八、撤销/重做系统

#### 8.1 命令记录

```javascript
// 每次命令执行后，记录到 undoStack
{
    type: 'create',            // 操作类型
    name: 'A',                 // 操作对象
    snapshot: {                // 快照（用于撤销）
        type: 'point',
        data: { x: 0, y: 0, z: 0 },
        style: { ... },
        parents: []
    }
}
// 或
{
    type: 'delete',
    name: 'A',
    snapshot: { ... }          // 被删前的快照
}
// 或
{
    type: 'modify',
    name: 'A',
    before: { x: 0, y: 0, z: 0 },
    after:  { x: 1, y: 0, z: 0 }
}
```

#### 8.2 撤销/重做逻辑

```javascript
function undo() {
    const record = undoStack.pop();
    switch (record.type) {
        case 'create':  // 撤销创建 = 删除
            deleteObject(record.name);
            break;
        case 'delete':  // 撤销删除 = 恢复
            restoreObject(record.name, record.snapshot);
            break;
        case 'modify':  // 撤销修改 = 恢复旧值
            updatePosition(record.name, record.before);
            break;
    }
    redoStack.push(record);
}

function redo() {
    const record = redoStack.pop();
    // 反向操作...
    undoStack.push(record);
}
```

### 十三、事件/通信系统

#### 13.1 事件总线设计

采用发布订阅模式实现模块间解耦通信：

```javascript
// EventBus（全局单例）
class EventBus {
    constructor() {
        this.listeners = new Map();
    }

    on(event, callback) {
        if (!this.listeners.has(event)) {
            this.listeners.set(event, new Set());
        }
        this.listeners.get(event).add(callback);
        return () => this.off(event, callback);  // 返回取消订阅函数
    }

    off(event, callback) {
        this.listeners.get(event)?.delete(callback);
    }

    emit(event, data) {
        this.listeners.get(event)?.forEach(cb => cb(data));
    }
}

export const bus = new EventBus();
```

#### 13.2 事件类型定义

```javascript
// 对象生命周期事件
'object:created'    { name, obj }              // 对象创建
'object:updated'    { name, obj, changes }     // 对象更新（坐标/样式）
'object:deleted'    { name, obj }              // 对象删除
'object:restored'   { name, obj }              // 对象恢复（撤销删除）

// 选择事件
'select:changed'    { selected, previous }     // 选中对象变更
'select:cleared'    {}                         // 取消所有选中

// 拖拽事件
'drag:start'        { name, position }         // 开始拖拽
'drag:move'         { name, position }         // 拖拽中
'drag:end'          { name, position }         // 结束拖拽

// 命令事件
'command:executed'  { ast, result }            // 命令执行完成
'command:error'     { ast, error }             // 命令执行失败
'command:undo'      { record }                 // 撤销
'command:redo'      { record }                 // 重做

// 渲染事件
'render:dirty'      {}                         // 需要重绘
'render:complete'   {}                         // 渲染完成

// 工具事件
'tool:changed'      { current, previous }      // 工具切换
```

#### 13.3 状态变更通知流程

```
用户拖拽点 A
    ↓
drag:move 事件
    ↓
ObjectStore 更新 A 的坐标
    ↓
ObjectStore 查询依赖图，获取受影响对象 ['E', 'seg1', 'polygon1']
    ↓
按拓扑序更新每个依赖对象
    ↓
对每个更新的对象 emit 'object:updated'
    ↓
GeomRenderer 监听 'object:updated'，更新对应的 Three.js 对象
    ↓
emit 'render:dirty'
    ↓

```

```javascript
// 深度合并
function deepMerge(target, source) {
    for (const key of Object.keys(source)) {
        if (source[key] && typeof source[key] === 'object' && !Array.isArray(source[key])) {
            target[key] = target[key] || {};
            deepMerge(target[key], source[key]);
        } else {
            target[key] = source[key];
        }
    }
    return target;
}
```

#### 17.4 LocalStorage 自动保存

```javascript
// storage.js - 自动保存管理器

const STORAGE_KEY = 'geometry-tool-autosave';
const SAVE_INTERVAL = 2000;  // 2 秒节流

class AutoSave {
    constructor(store) {
        this.store = store;
        this.timer = null;

        // 监听对象变更事件
        bus.on('object:created', () => this.scheduleSave());
        bus.on('object:updated', () => this.scheduleSave());
        bus.on('object:deleted', () => this.scheduleSave());
    }

    // 节流保存
    scheduleSave() {
        if (this.timer) clearTimeout(this.timer);
        this.timer = setTimeout(() => this.save(), SAVE_INTERVAL);
    }

    // 保存到 localStorage
    save() {
        const data = {
            name: this.store.projectName || '未命名项目',
            updated: new Date().toISOString(),
            objects: this.store.serialize()
        };
        localStorage.setItem(STORAGE_KEY, JSON.stringify(data));
    }

    // 从 localStorage 恢复
    load() {
        const saved = localStorage.getItem(STORAGE_KEY);
        if (saved) {
            try {
                const data = JSON.parse(saved);
                this.store.deserialize(data.objects);
                return data;
            } catch (e) {
                console.warn('自动保存数据损坏，已忽略');
                return null;
            }
        }
        return null;
    }

    // 清除自动保存
    clear() {
        localStorage.removeItem(STORAGE_KEY);
    }
}
```

```javascript
        reader.onload = (e) => {

    });
}

    // 保存按钮

        input.accept = '.json';
```

**Vite 插件（server/vite-plugin-api.js）：**

```javascript
import Database from 'better-sqlite3';
import { resolve } from 'path';

export default function apiPlugin() {
    const dbPath = resolve(__dirname, '../data/geometry.db');
    const db = new Database(dbPath);

    // 初始化表
    db.exec(`
        CREATE TABLE IF NOT EXISTS projects (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            name TEXT NOT NULL DEFAULT '未命名项目',
            data TEXT NOT NULL,
            created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
            updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
        )
    `);

    // 预编译 SQL
    const stmts = {
        list:    db.prepare('SELECT id, name, created_at, updated_at FROM projects ORDER BY updated_at DESC'),
        get:     db.prepare('SELECT * FROM projects WHERE id = ?'),
        insert:  db.prepare('INSERT INTO projects (name, data) VALUES (?, ?)'),
        update:  db.prepare('UPDATE projects SET name = ?, data = ?, updated_at = CURRENT_TIMESTAMP WHERE id = ?'),
        delete:  db.prepare('DELETE FROM projects WHERE id = ?'),
    };

    return {
        name: 'vite-plugin-api',
        configureServer(server) {
            // 解析 JSON body
            server.middlewares.use('/api', async (req, res, next) => {
                if (req.method === 'POST' || req.method === 'PUT') {
                    const chunks = [];
                    for await (const chunk of req) chunks.push(chunk);
                    req.body = JSON.parse(Buffer.concat(chunks).toString());
                }
                next();
            });

            // GET /api/projects - 项目列表
            server.middlewares.use('/api/projects', (req, res) => {
                if (req.method === 'GET') {
                    const projects = stmts.list.all();
                    res.setHeader('Content-Type', 'application/json');
                    res.end(JSON.stringify(projects));
                }
            });

            // POST /api/projects - 创建项目
            server.middlewares.use('/api/projects', (req, res) => {
                if (req.method === 'POST') {
                    const { name, data } = req.body;
                    const result = stmts.insert.run(name || '未命名项目', JSON.stringify(data));
                    res.setHeader('Content-Type', 'application/json');
                    res.end(JSON.stringify({ id: result.lastInsertRowid }));
                }
            });

            // PUT /api/projects/:id - 更新项目
            server.middlewares.use('/api/projects/', (req, res) => {
                const id = req.url.split('/')[0];
                if (req.method === 'PUT') {
                    const { name, data } = req.body;
                    stmts.update.run(name, JSON.stringify(data), id);
                    res.setHeader('Content-Type', 'application/json');
                    res.end(JSON.stringify({ success: true }));
                }
            });

            // DELETE /api/projects/:id - 删除项目
            server.middlewares.use('/api/projects/', (req, res) => {
                const id = req.url.split('/')[0];
                if (req.method === 'DELETE') {
                    stmts.delete.run(id);
                    res.setHeader('Content-Type', 'application/json');
                    res.end(JSON.stringify({ success: true }));
                }
            });
        }
    };
}
```
