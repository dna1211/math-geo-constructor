# 几何命令生成规则

> 将文字描述转换为 3D 几何构造命令的规则文档
> 版本：v0.3（优化步骤解析逻辑，支持混合写法）

---

## 一、坐标系约定

- **右手坐标系**：X 向右，Y 向前（屏幕里），Z 向上
- **单位**：无固定单位，默认边长为 1
- **原点**：默认居中于原点 (0, 0, 0)

---

## 二、命令语法速查

### 构造类

| 命令 | 语法 | 说明 |
|------|------|------|
| Point | `Name = Point(x, y, z)` 或 `Name = (x, y, z)` | 创建点 |
| Segment | `Segment(A, B)` | 线段（两点间） |
| Line | `Line(A, B)` | 无限直线 |
| Ray | `Ray(A, B)` | 射线（从 A 过 B 延伸） |
| Triangle | `Triangle(A, B, C)` | 三角形面 |
| Polygon | `Polygon(A, B, C, ...)` | 多边形面（≥3 点） |
| Plane | `Plane(A, B, C)` 或 `Plane(a, b, c, d)` | 平面 |

### 计算类

| 命令 | 语法 | 说明 |
|------|------|------|
| Midpoint | `M = Midpoint(A, B)` | 中点（自动跟随父对象） |
| Fold | `F = Fold(P, Line(A,B), angle)` | 绕轴旋转（角度制） |
| Reflect | `R = Reflect(P, Line(A,B))` 或 `Reflect(P, Plane(...))` | 对称/镜像 |
| Distance | `D = Distance(A, B)` | 两点距离 |

### 高级构造

| 命令 | 语法 | 说明 |
|------|------|------|
| RegularPolygon | `RegularPolygon(Segment(A,B), n, Plane(X,Y,Z), angle)` | 正 n 边形 |
| EquilateralTriangle | `EquilateralTriangle(Segment(A,B), Plane(X,Y,Z), angle)` | 正三角形 |

### 样式类

| 命令 | 语法 | 说明 |
|------|------|------|
| Color | `Color(A, "#ff0000")` 或 `Color(A, '#ff0000')` | 设置颜色（支持单/双引号） |
| Dash | `Dash(A, true)` | 虚线 |
| Opacity | `Opacity(A, 0.5)` | 透明度（0-1） |
| Hide | `Hide(A)` | 隐藏 |
| Show | `Show(A)` | 显示 |
| Label | `Label(A, "文字")` | 自定义标签 |

### 操作类

| 命令 | 语法 | 说明 |
|------|------|------|
| Delete | `Delete(A)` | 删除对象 |
| Undo | `Undo` | 撤销 |
| Redo | `Redo` | 重做 |
| Clear | `Clear` | 清空全部 |
| Grid | `Grid(true/false)` | 网格开关 |
| Axis | `Axis(true/false)` | 坐标轴开关 |

### 步骤控制

| 语法 | 说明 |
|------|------|
| `---step 步骤名称---` | 标记新步骤的开始 |

**步骤功能说明：**
- 使用 `---step 名称---` 将命令序列分割为多个步骤
- 步骤内的命令会一次性执行
- 可以通过步骤控制面板逐步展示构造过程
- 显示当前步骤及之前所有步骤的对象

**步骤标记语义：**
- `---step 名称---` 是"标题式"标记，标记**接下来的命令**属于这个步骤
- 支持两种写法：
  - **写法1**：第一步写步骤名（推荐，语义更清晰）
  - **写法2**：第一步不写步骤名（自动归入"准备"步骤）

## 三、命名规则

| 对象类型 | 前缀 | 示例 |
|----------|------|------|
| 点 | 大写字母 | `A`, `B`, `P1`, `V1` |
| 线段 | `seg` + 数字 | `seg1`（自动命名） |
| 直线 | `line` + 数字 | `line1`（自动命名） |
| 射线 | `ray` + 数字 | `ray1`（自动命名） |
| 三角形 | `tri` + 数字 | `tri1`（自动命名） |
| 多边形 | `poly` + 数字 | `poly1`（自动命名） |
| 平面 | `plane` + 数字 | `plane1`（自动命名） |
| 中点 | 大写字母 | `M`, `M1` |
| 正多边形顶点 | `v` + 边数 + `_` + 数字 | `v5_1`（正五边形顶点） |

**规则：**
- 用户显式赋值时使用用户指定的名称：`A = Point(0,0,0)`
- 不赋值时系统自动命名：`Segment(A, B)` → 自动命名为 `seg1`
- 正多边形的顶点由系统自动命名和创建

## 四、转换规则

### 4.1 基本几何体

#### 点

| 描述 | 命令 |
|------|------|
| "原点" / "在原点放一个点" | `O = Point(0, 0, 0)` |
| "点 (1, 2, 3)" | `A = Point(1, 2, 3)` |
| "X 轴上距离原点 3 的点" | `A = Point(3, 0, 0)` |
| "Z 轴上方 5 个单位" | `A = Point(0, 0, 5)` |

#### 线段

| 描述 | 命令 |
|------|------|
| "连接 A 和 B 的线段" | `Segment(A, B)` |
| "从原点到 (1,0,0) 的线段" | `O = Point(0,0,0)` 然后 `A = Point(1,0,0)` 然后 `Segment(O, A)` |

#### 三角形

| 描述 | 命令 |
|------|------|
| "三角形 ABC" | `Triangle(A, B, C)` |
| "边长为 2 的正三角形（XY 平面）" | `A = Point(0,0,0)` `B = Point(2,0,0)` `C = Point(1,1.732,0)` `Triangle(A,B,C)` |

#### 多边形

| 描述 | 命令 |
|------|------|
| "四边形 ABCD" | `Polygon(A, B, C, D)` |
| "正方形（边长 1，XY 平面）" | `A=(0,0,0) B=(1,0,0) C=(1,1,0) D=(0,1,0)` `Polygon(A,B,C,D)` |

#### 平面

| 描述 | 命令 |
|------|------|
| "过 A B C 三点的平面" | `Plane(A, B, C)` |
| "XY 平面" | `Plane(0, 0, 1, 0)` （方程 z=0） |
| "XZ 平面" | `Plane(0, 1, 0, 0)` （方程 y=0） |
| "YZ 平面" | `Plane(1, 0, 0, 0)` （方程 x=0） |

### 4.2 计算与变换

| 描述 | 命令 |
|------|------|
| "A 和 B 的中点" | `M = Midpoint(A, B)` |
| "A 关于线 L 对称的点" | `A' = Reflect(A, L)` |
| "A 绕轴旋转 90 度" | `A' = Fold(A, Axis, 90)` |
| "A 到 B 的距离" | `D = Distance(A, B)` |

### 4.3 样式控制

| 描述 | 命令 |
|------|------|
| "把 A 变成红色" | `Color(A, "#ff0000")` |
| "设为虚线" | `Dash(A, true)` |
| "半透明" | `Opacity(A, 0.3)` |
| "隐藏 A" | `Hide(A)` |

### 4.4 复合图形（分步构建）

对于复杂图形，需要先创建点，再组合：

**正方形：**
```
A = Point(0, 0, 0)
B = Point(1, 0, 0)
C = Point(1, 1, 0)
D = Point(0, 1, 0)
Polygon(A, B, C, D)
```

**正五边形（使用 RegularPolygon）：**
```
A = Point(0, 0, 0)
B = Point(1, 0, 0)
S = Segment(A, B)
P = Plane(0, 0, 1, 0)
RegularPolygon(S, 5, P, 0)
```

### 4.5 使用步骤分步展示

对于教学或证明场景，可以使用步骤功能逐步展示构造过程：

**基本语法：**
```
---step 步骤名称---
```

**写法1：第一步写步骤名（推荐）**
```
---step 底面顶点---
A = Point(0, 0, 0)
B = Point(1, 0, 0)
C = Point(1, 1, 0)
D = Point(0, 1, 0)
---step 底面---
Polygon(A, B, C, D)
---step 顶面顶点---
E = Point(0, 0, 1)
F = Point(1, 0, 1)
G = Point(1, 1, 1)
H = Point(0, 1, 1)
---step 顶面---
Polygon(E, F, G, H)
```

解析结果：
| 步骤 | 名称 | 命令 |
|------|------|------|
| 0 | 底面顶点 | A, B, C, D 点 |
| 1 | 底面 | Polygon |
| 2 | 顶面顶点 | E, F, G, H 点 |
| 3 | 顶面 | Polygon |

**写法2：第一步不写步骤名（自动归入"准备"）**
```
A = Point(0, 0, 0)
B = Point(1, 0, 0)
C = Point(1, 1, 0)
D = Point(0, 1, 0)
---step 底面---
Polygon(A, B, C, D)
---step 顶面顶点---
E = Point(0, 0, 1)
F = Point(1, 0, 1)
G = Point(1, 1, 1)
H = Point(0, 1, 1)
---step 顶面---
Polygon(E, F, G, H)
```

解析结果：
| 步骤 | 名称 | 命令 |
|------|------|------|
| 0 | 准备 | A, B, C, D 点 |
| 1 | 底面 | Polygon |
| 2 | 顶面顶点 | E, F, G, H 点 |
| 3 | 顶面 | Polygon |

**示例：证明题的分步展示**
```
---step 基础四边形---
A = Point(0, 0, 0)
B = Point(4, 0, 0)
C = Point(4, 3, 0)
D = Point(0, 3, 0)
---step 底面---
Polygon(A, B, C, D)
---step 对角线---
segAC = Segment(A, C)
segBD = Segment(B, D)
Dash(segAC, true)
Dash(segBD, true)
---step 中点---
M = Midpoint(A, C)
N = Midpoint(B, D)
---step 中点连线---
segMN = Segment(M, N)
Color(segMN, "#44cc44")
```

## 六、规则约定（待完善）

### 6.0 对象命名与样式规则（强制）

**1. 每个几何对象必须有名称**

所有几何对象在创建时必须赋名称，以便后期操作（设置样式、删除等）。

| 写法 | 说明 |
|------|------|
| `A = Point(0, 0, 0)` | ✅ 正确，显式命名 |
| `seg12 = Segment(P, C)` | ✅ 正确，显式命名 |
| `line1 = Line(A, B)` | ✅ 正确，显式命名 |
| `Segment(A, B)` | ❌ 避免，自动命名不直观，后期操作不便 |

**用途示例：**
```
seg12 = Segment(P, C)
Dash(seg12, true)       -- 可按名称操作
Color(seg12, "#ff0000") -- 可按名称改色
LineWidth(seg12, 3)     -- 可按名称改线宽
```

**2. 辅助线默认虚线**

生成命令时，需根据题目语义自动判断线条性质：

| 线条性质 | 样式 | 判断依据 | 示例 |
|----------|------|----------|------|
| 几何体棱边 | **实线** | 构成几何体轮廓的边 | 正方体的 12 条棱、三角形的 3 条边 |
| 辅助线 | **虚线** | 为解题额外添加的线段 | 中点连线、对角线、证明用辅助线 |
| Line（无限直线） | **虚线** | 默认虚线 | 辅助轴、参考线 |
| Ray（射线） | **虚线** | 默认虚线 | 方向参考线 |

**判断规则：**

- **实线**：题目中明确给出的边、构成几何体表面的线段
  - 例：正方体 ABCD-EFGH 的棱 AB、BC、CD... → 实线
  - 例：三角形 ABC 的边 AB、BC、CA → 实线

- **虚线**：为解题或证明额外构造的线段
  - 例：取中点 M，连接 AM → `Dash(AM, true)`
  - 例：作对角线 BD → `Dash(BD, true)`
  - 例：证明用的辅助连线 → `Dash(seg1, true)`

**示例：**
```
# 正方体棱边 — 实线（默认）
seg1 = Segment(A, B)
seg2 = Segment(B, C)

# 辅助线 — 虚线
M = Midpoint(A, C)
seg3 = Segment(B, M)
Dash(seg3, true)        # 中点连线为辅助线，虚线

# 辅助对角线
seg4 = Segment(A, C)
Dash(seg4, true)        # 对角线为辅助线，虚线
```

**3. 用对角线交点表示几何中心**

正方形、矩形、平行四边形等的中心，通过对角线交点获得，而非直接 `Midpoint`。

```
# ❌ 不推荐：看不出几何意义
M = Midpoint(A, C)

# ✅ 推荐：画出对角线，交点即中心
segAC = Segment(A, C)
Dash(segAC, true)
segBD = Segment(B, D)
Dash(segBD, true)
M = Midpoint(A, C)      # AC 与 BD 的交点，正方形中心
```

同理，证明中涉及中位线时，应画出完整三角形，使中位线关系一目了然：

```
# 画出 △PAC，EM 为中位线的几何关系直观可见
triPAC = Triangle(P, A, C)
E = Midpoint(P, A)
M = Midpoint(A, C)
segEM = Segment(E, M)
Dash(segEM, true)
```

**4. 证明目标颜色标注**

对题目中需要证明的线面关系做颜色标注，使目标结构一目了然：

| 目标类型 | 颜色 | 说明 |
|----------|------|------|
| 要证的线 | 红色 `#ff4444` | 如"证明 PC // 平面 BDE"中的 PC |
| 要证的面 | 蓝色半透明 `#4488ff` opacity 0.2 | 如"证明 PC // 平面 BDE"中的平面 BDE |
| 桥梁线 | 绿色 `#44cc44` | 证明中起连接作用的辅助线（如中位线 EM） |

```
Color(segPC, "#ff4444")         # 要证的线 — 红色
triBDE = Triangle(B, D, E)      # 要证的面
Color(triBDE, "#4488ff")
Opacity(triBDE, 0.2)            # 蓝色半透明
Color(segEM, "#44cc44")         # 桥梁线 — 绿色
```

### 6.7 步骤使用规则（强制）

**1. 步骤标记语法**

使用 `---step 名称---` 标记新步骤的开始，该标记必须独占一行。

```
---step 步骤名称---
```

**2. 步骤标记语义**

`---step 名称---` 是"标题式"标记，标记**接下来的命令**属于这个步骤。

**支持两种写法：**

| 写法 | 说明 | 示例 |
|------|------|------|
| 写法1（推荐） | 第一步写步骤名 | `---step 底面---` → 命令 |
| 写法2 | 第一步不写步骤名 | 命令 → `---step 下一步骤---` → 命令 |

**写法1示例：**
```
---step 底面顶点---
A = Point(0, 0, 0)
B = Point(1, 0, 0)
C = Point(1, 1, 0)
D = Point(0, 1, 0)
---step 底面---
Polygon(A, B, C, D)
```
结果：步骤0"底面顶点"包含4个点，步骤1"底面"包含Polygon

**写法2示例：**
```
A = Point(0, 0, 0)
B = Point(1, 0, 0)
C = Point(1, 1, 0)
D = Point(0, 1, 0)
---step 底面---
Polygon(A, B, C, D)
```
结果：步骤0"准备"包含4个点，步骤1"底面"包含Polygon

**3. 步骤划分原则**

| 场景 | 步骤划分建议 |
|------|--------------|
| 几何体构造 | 按构造逻辑划分：基础点 → 底面 → 顶面 → 棱边 → 侧面 |
| 证明题 | 按证明思路划分：已知条件 → 辅助线 → 关键连线 → 证明目标 |
| 教学演示 | 按教学节奏划分：每步展示一个概念或操作 |

**4. 步骤命名规范**

- 使用简洁明了的名称，概括本步骤内容
- 推荐使用中文，便于理解
- 示例：
  - ✅ `---step 底面四点---`
  - ✅ `---step 作对角线---`
  - ✅ `---step 取中点---`
  - ❌ `---step 步骤1---`（不够具体）
  - ❌ `---step 点和线---`（过于模糊）

**5. 步骤粒度建议**

- 每个步骤包含 2-5 条命令为宜
- 避免单步过多命令（失去分步意义）
- 避免单步仅一条命令（过于碎片化）

**6. 步骤与辅助线**

辅助线应在独立步骤中创建，便于区分主体结构和辅助构造：

```
---step 基础四边形---
A = Point(0, 0, 0)
B = Point(4, 0, 0)
C = Point(4, 3, 0)
D = Point(0, 3, 0)
---step 底面---
Polygon(A, B, C, D)

# 辅助线（独立步骤）
---step 对角线---
segAC = Segment(A, C)
segBD = Segment(B, D)
Dash(segAC, true)
Dash(segBD, true)
```

**7. 步骤与证明目标**

证明目标应在最后步骤中展示，并使用颜色标注：

```
---step 证明目标---
segPC = Segment(P, C)
Color(segPC, "#ff4444")
triBDE = Triangle(B, D, E)
Color(triBDE, "#4488ff")
Opacity(triBDE, 0.2)
```

### 6.8 输出文件规范

每道大题生成 **一个命令文件**（`xxx-commands.txt`），内部按小题分段。

**文件结构：**
```
# === (1) 小题标题 ===
（完整的可执行命令，含所有基础结构 + 本题专用内容）

# === (2) 小题标题 ===
（完整的可执行命令，含所有基础结构 + 本题专用内容）
```

**规则：**
- 一个大题 = 一个文件，不拆分多个文件
- 每小题的命令段是 **完整独立** 的（含全部基础点/线/面 + 本题辅助线）
- 用户每做一小题前先 Clear，再复制该段完整命令执行
- 只含可执行命令，不用 `#` 写解释（解释放单独的解题文件）
- 段间用 `# === (N) 标题 ===` 分隔

**步骤标记的使用：**
- 命令文件中应包含 `---step---` 标记，便于逐步展示
- 步骤划分应符合解题逻辑和教学节奏
- 每小题的步骤独立编号（从1开始）

**示例（带步骤的命令文件）：**
```
# === (1) 证明 AC ⊥ BD ===
---step 基础四边形---
A = Point(0, 0, 0)
B = Point(4, 0, 0)
C = Point(4, 3, 0)
D = Point(0, 3, 0)
---step 底面---
Polygon(A, B, C, D)
---step 对角线---
segAC = Segment(A, C)
segBD = Segment(B, D)
Dash(segAC, true)
Dash(segBD, true)
---step 证明目标---
Color(segAC, "#ff4444")
Color(segBD, "#ff4444")

# === (2) 求对角线长度 ===
...
```

**解题文件（`xxx-解题.txt`）：**
- 纯文字，含完整解题过程和证明
- 不含命令，用于对照理解

### 6.8 数学题条件解析模板

**输入格式：** 用户给出题目条件描述

**解析步骤：**
1. 识别所有几何对象（点、线、面、体）
2. 识别约束条件（平行、垂直、等边、中点等）
3. ⚠️ **检查用户是否指定了坐标系**（原点、底面、X/Y 轴方向）
   - **已指定** → 按用户指定建立坐标系
   - **未指定** → **必须提醒用户补充，不可自行猜测**
4. 计算各点坐标
5. 按依赖顺序生成命令（先点 → 再线/面 → 最后辅助线）

**⚠️ 强制规则：坐标系不可猜测**

用户必须明确指定以下至少一项：
- 原点位置（如"D 为原点"、"A 为原点"）
- 底面所在平面（如"ABCD 在 XY 平面"）
- 坐标轴方向（如"DA 为 X 轴"、"AB 为 Y 轴"）

若用户未指定，必须回复：
> 请指定坐标系：哪个点为原点？哪条边为 X 轴或 Y 轴？或底面在哪个平面？

**条件→坐标映射：**

| 条件 | 处理方式 |
|------|----------|
| "正方形 ABCD" | 计算四点坐标，用 Polygon |
| "等边三角形 PAD" | 边长 L，高 L√3/2，确定第三点坐标 |
| "平面 PAD ⊥ 平面 ABCD" | ABCD 在 XY 平面，PAD 法向量在 XY 平面内 |
| "AD = 2" | 确定两点间距离为 2 |
| "E 为 PA 中点" | E = Midpoint(P, A) |
| "证明 PC // 平面 BDE" | 取辅助点 M，证明 PC // EM，EM ⊂ 平面 BDE |
